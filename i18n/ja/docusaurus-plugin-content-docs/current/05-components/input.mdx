# Input <Badge text='ready' color='green'></Badge> <StorybookLink path='/story/forms-input--simple' />

ユーザーがテキストを記入または編集できる基本的なコンポーネントです。

## 使用する場面

フォーム内で入力欄を使用し、ユーザーがテキストを入力、選択、検索できるようにします。入力欄は通常フォーム内に配置されますが、モーダル、検索、カードの一部としても使用できます。

### フォームでの使用

入力欄は**Field**コンポーネントと併用し、ラベルや説明を提供する必要があります。

```tsx live
<Field label="Username">
  <Input placeholder="Enter username" />
</Field>
```

## 実装/オプション

### プレースホルダーテキスト

入力欄にはプレースホルダーテキストを設定できます。ただし、プレースホルダーテキストは実際のテキストと混同されやすいため、控えめに使用してください。混同されると、ユーザーが入力必須項目を見逃し、エラー率が高くなる可能性があります。Fieldコンポーネントと併用する場合は、プレースホルダーテキストではなくラベルと説明を使用してください。

### 入力欄内の接頭辞または接尾辞

入力欄の先頭または末尾にアイコンなどの要素を配置できます。これにより、虫眼鏡アイコン付きの検索フィールドなどの入力欄を作成できます。

```tsx live
<Input prefix={<Icon name="search" />} placeholder="Search dashboards" />
```

```tsx live
<Input suffix={<Icon name="plus-circle" />} />
```

### 入力欄の前後のアドオン

一般的にボタンなどのアクセサリを入力欄の前後に配置できます。

```tsx live
<Input addonAfter={<Button variant="secondary">Load</Button>} />
```

## タイプと動作

### テキスト

テキスト入力欄では、ユーザーはテキスト、数字、特殊文字を入力できます。

### 数値

数値入力欄では、ユーザーは数字のみを入力できます。

### パスワード

パスワード入力欄では、ユーザーが入力した文字がマスク表示されます。

### 状態

#### 無効

入力欄の無効状態は、提供された内容に修正が必要であるか、ユーザーが進む前に追加の内容が必要であることを示します。

#### 読み込み中

状況によっては、入力欄が読み込み中であることを示すことができます。

#### 無効化

ユーザーの権限や入力欄がデータソースと同期されている場合など、入力欄を無効化状態にできます。
