# 削除 <Badge text='draft' color='orange'></Badge>

Grafana内での削除は、必要な摩擦の量（参照：[デザイン原則：適度な摩擦](/foundations/design-principles#tasteful-friction)）と削除対象の重要性に基づいて異なる多層的なパターンです。

# 階層

適切なパターンを選択するために定義とユースケースを確認してください。

### **インライン削除**

リスクの低いコンテンツを簡単に削除する方法で、ユーザーが再追加しやすい場合に使用されます。通常、**trash-alt**、**times-circle**、または**times**アイコン付きのIconButtonで使用され、確認ステップはありません。行やリスト項目の削除などに使用されます。

### **入力して削除（削除ダイアログ）**

最も摩擦が大きいパターン：誤った削除が壊滅的な結果をもたらす可能性がある場合に使用します。このパターンでは、ユーザーが何を破壊しようとしているのか（そしてそれが何なのか）を完全に認識し、ボタンクリック以上の明示的な確認を要求します。変更が波及効果を持つ場合に有用です。
