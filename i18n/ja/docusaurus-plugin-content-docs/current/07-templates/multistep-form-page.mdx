## 概要

マルチステップフォーム（ウィザード）パターンは、複雑な設定やセットアップ関連のフローをステップに分割することで、ユーザーが確実に完了できるように支援します。情報やステップを機能別または意味的にグループ化することで認知負荷を軽減し、エラーからの迅速な回復を可能にするとともに、進捗を保存して中断した場所に簡単に戻れるようにすることでセッションの中断を許容します。

**注記:** 大規模なセットアップや追加ライブラリを避けるため、このテンプレートではステップナビゲーションに `window.location.hash` を使用しています。Grafanaでは、ルーティングナビゲーションを処理するために `@grafana/runtime` の `locationService` を使用しています。

## 使用方法

### 使用すべき場合

ウィザードパターンは、以下のようなインタラクションに使用すべきです：

- 複雑または長い処理
- 論理的なステップに分割可能な場合
- 特定の順序で実行する必要がある場合
- 明確な開始と終了がある場合（`追加`で開始、`保存`で終了）
- オブジェクトの作成や設定の変更など、特定のアクションを完了するための処理

### 使用すべきでない場合

- 短いフォーム
- ステップに論理的に分割できないフォーム
- プロダクトツアー
