# デザイン原則 <Badge text='draft' color='orange'></Badge>

デザイン原則は、組織内のデザイナーや開発者が、意思決定者に関わらず同じ方法でデザイン決定を行えるようにするものです。製品やデザインの決定をこれらの一貫した方法に根ざすことで、さまざまな問題に対して同じ視点を持つことで一貫性を保つのに役立ちます。

## タスク優先

ユーザーの主要なタスクや焦点にデザイン決定を基づかせ、ユーザーがそれらのタスクを簡単に完了できるようにします。無関係なUIの雑多さやユーザーフローによって圧倒されないようにします。

**私たちのUIは、ユーザーがタスクを完了するのを助けるべきであり、妨げるべきではありません。**

## 適切な摩擦

摩擦は、リスクや損害の可能性が高いアクションに対して高くするべきです。変更の保存を考えてみましょう。チームの名前を保存するだけの場合：リスクは低く、実際の副作用もなく、簡単に修正できます。しかし、多くの人が使用するダッシュボードやパネルを完全に上書きする「保存」の場合、他のダッシュボード、レポート、アラートに波及効果があるため、ミスによる損害の可能性ははるかに高くなります。一方のケースでは非常に少ない摩擦で済みますが、もう一方のケースでは、ユーザーは自分が何をしているのかを非常に明確にし、そのアクションの結果を完全に認識している必要があります。

**UIの摩擦は、ユーザーのアクションの重大性とリスクに合わせるべきです。**

## ユーザーのためのシンプルな解決策

シンプルな解決策とは、それ自体が複雑さを欠いていることを意味するわけではありません。通常、UIは、ユーザーとシステムの間で、誰がより重い複雑さの負担を負うかを交渉し、妥協します。ユーザーにとってのシンプルな解決策は、UXデザイナーや開発者にとってはより多くの作業を意味するかもしれませんが、ユーザーにとってはよりシンプルな体験とUIをもたらします。

**シンプルな解決策とは、簡単な解決策を意味するわけではありません。アクションの複雑さはシステムが引き受けます。**

## 再利用をデフォルトにする

製品全体で類似したパターンを一致させることで、ユーザーが製品スイートをナビゲートする際に期待される体験を提供します。可能な限りコンポーネントやパターンを再利用することで、UIの分岐を減らし、一貫したインタラクションと体験を作り出し、ユーザーにとってイライラしたり驚かせたりする状況を作る可能性を低くします。

**一貫したUIとパターンは、期待されるインタラクションと動作を作り出し、より良い体験をもたらします。**
