# アラート <Badge text='ready' color='green'></Badge> <StorybookLink path='/story/overlays-alert-inlinebanner--basic'/>

アラートは、ユーザーに重要なメッセージや通知を表示するための基本的な要素です。

## 使用する場面

アラートは、更新、変更、または問題についてユーザーに通知するために使用します。ユーザーにフィードバックを提供し、情報を伝え、中断を最小限に抑えることが重要です。アラートには、タスク生成とシステム生成の2つの異なる使用ケースがあります。

## 使用方法

### タスク生成

タスク生成アラートは、ユーザーの行動に基づいてトリガーされ、迅速で直接的なフィードバックを提供します。このタイプのアラートは、ユーザーが現在作業している領域の近くに配置する必要があります。

### システム生成

システム生成アラートは、ユーザーの行動とは関係なく、システムによってトリガーされます。システムの状態や他の場所で発生したイベントの説明を提供するために、これらのアラートを使用します。

## 実装 / オプション

### コンテンツ

#### タイトル

アラートのタイトルは必須です。簡潔で、人間中心の、専門用語を使わないようにし、ユーザーが簡単に理解できるようにする必要があります。

#### 説明

説明文はオプションで、控えめに使用する必要があります。説明文は短く、アラートの理由に焦点を当て、ユーザーがブロックされないようにするための解決策や次のステップを含めるようにします。

#### コンテンツガイドライン（書き方）

アラートのメッセージを作成する際には、常にシンプルで簡潔な言語を使用することが重要です。専門用語やシステムエラーをアラートの内容に使用しないでください。ユーザーに次のステップや可能な解決策を提供します。書き方の詳細については、[UXライティングガイドライン](https://grafana.com/docs/writers-toolkit/write/style-guide/ux-writing/)を参照してください。

#### 深刻度

アラートには4つの異なるメッセージタイプがあります：

- _情報_ - ユーザーに情報を提供し、現在の行動を中断しない
- _成功_ - タスクが期待通りに完了したことを確認する
- _警告_ - 望ましくない行動をユーザーに通知する
- _エラー_ - 期待通りに行動が行われなかった場合、通常何かが失敗し、ユーザーが行動を取る必要がある

#### コール・トゥ・アクション

アラート（特にインラインアラート）には、コール・トゥ・アクション（CTA）を含めることができます。特に、アラートが表示されている理由を修正または改善するためにユーザーが取れる行動がある場合です。アラート内にCTAを提供する場合は、ラベルを簡潔に保ち、ユーザーが取れる行動が明確であることを確認し、ラベルを短く（数語程度）に保つようにします。Grafanaの[UXライティングガイドライン](https://grafana.com/docs/writers-toolkit/write/style-guide/ux-writing/)を参照してください。

## タイプと動作

Grafanaでは、アラートを2つの方法で使用します：インラインとトーストです。これら2つの異なるタイプのアラートは、異なる目的と動作を持っています。適切なものを使用するために、違いを理解することが重要です。

### インライン

インラインアラートは、システムおよびユーザーによってトリガーされたイベントをユーザーに通知します。ユーザーが現在作業しているメインコンテンツ領域の上部または下部に表示されます。これにより、コンテキストが維持され、理解が促進されます。これらのメッセージはタスク生成であり、直接的で文脈に沿ったフィードバックを提供します。

```tsx live
<Alert title="Inline alert">
  <div>Child content that includes some alert details, like maybe what actually happened.</div>
</Alert>
```

### トースト

トーストは、短いメッセージを表示するための一時的なウィンドウ要素です。画面の右上隅に表示され、`X`をクリックするか、数秒後に自動的に消えます。

```tsx live
<Alert title="Toast" elevated>
  <div>Child content that includes some alert details, like maybe what actually happened.</div>
</Alert>
```

#### 動作

- トーストは短時間表示されます
- ユーザーがアラートを閉じられるように閉じる「X」アクションを含みます
- 複数が同時に表示される場合、最新のアラートは常に最上部に表示されます。古いトーストは最新のアラートの下に順番に積み重なります。
