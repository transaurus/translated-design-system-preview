# ナビゲーション <Badge text='ready' color='green'></Badge>

ナビゲーションパターンは、ユーザーがアプリケーション内をシームレスに移動できるようにするために重要です。Grafanaでは、メガメニューとブレッドクラムの2つの主要コンポーネントがナビゲーションにおいて重要な役割を果たします。これらのコンポーネントが効果的なナビゲーションパターンにどのように貢献するかを見ていきましょう。

## メガメニューナビゲーション

**概要:**  
Grafanaのメガメニューは、コア、プラグイン、その他のアプリを含むGrafana全体の主要なナビゲーションコンポーネントです。メガメニューには、開いた状態、閉じた状態、および「ドッキング」された開いた状態の3つの状態があります。開いた状態では、メニューはオーバーレイとして機能します。ドッキングされた状態では、ページコンテンツが狭まったビューポートに対応します。

**使用するタイミング:**  
メガメニューは、Grafanaの情報アーキテクチャのレベル1、2、および3へのアクセスを提供します。これ以上のレベルは、ページ上のタブ（レベル4）または見出し、ステッパー、その他のデザイン要素を使用したインライン要素（レベル5以上）で処理する必要があります。

### 使用ガイドライン

ユーザーがGrafanaをナビゲートする際に一貫性のある期待通りの体験を得られるようにするためには、以下の「推奨事項」と「禁止事項」に従うことが重要です。

**推奨事項:**

- すべての概要/カテゴリページにランディングページを利用する。
- すべてのエントリに対応するページまたは移動先を確保し、ブレッドクラムの動作を一貫させる。
- レベル1のコンテンツには必ずアイコンを追加する。
- メガメニューのすべてのエントリはセンテンスケース（文のケース）で記述する。
- Grafana全体で使用するラベルの標準化を検討し、ページタイトルには動詞ではなく名詞を使用する。
- メガメニューに配置するページの数を考慮する。
- ページタイトルの長さに注意し、切り詰めを避ける。
- Grafanaエコシステム内での一貫性を追求する。

**禁止事項:**

- ランディングページがないことを補うためにメガメニュー内でリダイレクトを行わない。
- **レベル1以外のコンテンツ**にアイコンを使用しない。
- クラウドアプリケーションをメガメニューのレベル1に配置しない。
- タイトルケースを使用しない。
- 外部リンクを含めない。
- レベル5以上を示すためにタブをネストしない。

## ブレッドクラムコンポーネントのドキュメント

Grafanaのトップナビゲーションブレッドクラムは、全体的なナビゲーション体験の重要な部分です。これらは2022-2023年のナビゲーション再設計の中で最も人気のある部分であり、ユーザー体験にとって一貫した均一な動作が不可欠です。

**使用するタイミング:**  
ブレッドクラムはページ階層によって自動生成されるため、すべてのページで使用されます。

### 使用ガイドライン:

ブレッドクラムは、ユーザーに製品空間内での現在地を示す重要な指標を提供します。ブレッドクラムが製品全体で一貫して動作することで、ユーザーは良好な体験を得られ、現在表示している製品の部分に関係なく自分の位置を理解できます。以下のガイドラインに従うことで、一貫した期待通りの体験を提供できます。

**推奨事項:**

- 情報アーキテクチャの階層を示すためにブレッドクラムを利用する。
- ブレッドクラムとして反映されるものには別のページを利用する。
- すべてのブレッドクラムエントリはセンテンスケースで記述する。
- Grafanaエコシステム内での一貫性を追求する。
- ページタイトルの長さに注意し、切り詰めを避ける。

**禁止事項:**

- ブレッドクラムの動作を履歴を反映するようにオーバーライドしない。
- ランディングページがないことを補うためにブレッドクラムをリダイレクトしない。
- タイトルケースを使用しない。

## 前の画面に戻る

Grafanaの技術的および情報アーキテクチャのため、ユーザーがある領域でアイテムを操作し、その作業結果を確認するためにリンクをたどると、別の領域（例えばSLO設定がアラートルールを作成したり、クラウドアプリがダッシュボードを作成したりする）に移動することがよくあります。さらに、クラウドアプリの体験を拡張機能を通じてコアGrafanaに組み込むほど、情報アーキテクチャの「ツリー」の一方の枝から非常に遠い別の枝へユーザーをリンクさせることになります。

このため、コンテキストの切り替えが発生し、Grafanaの情報アーキテクチャを完全に理解する必要があります。

ブレッドクラムは履歴ではなく階層構造のために予約されているため、ユーザーはブレッドクラムを使って「戻る」ことができません。さらに、ブラウザの「戻る」ボタンは時折役立つ程度であり、多くのユーザーはWebアプリケーションではブラウザのボタンを無視するように訓練されています。

「前の画面に戻る」は、このような状況に遭遇したユーザーに対して前のコンテキストを復元するヘルパーコンポーネントです。

**使用するタイミング:**
リンクがユーザーをGrafanaの情報アーキテクチャの離れた部分に送る場合は、いつでも「前の画面に戻る」を使用してください。このコンポーネントは当初クラウドアプリを念頭に置いて設計されましたが、Grafanaのコア製品全体でも役立ちます。

### 使用ガイドライン:

「前の画面に戻る」を使用して、ユーザーが元の場所に戻るためのナビゲーション支援を提供します。

開発者向けのドキュメントは[Developer portal](https://grafana.com/developers/plugin-tools/create-a-plugin/extend-a-plugin/add-return-to-previous-functionality)で利用可能です。
